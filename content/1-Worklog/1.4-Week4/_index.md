**Weekly Objectives**

- Learn how to use Amazon QuickSight to build professional dashboards that support data-driven decision-making.
- Understand the various analytics services available on AWS and select the most appropriate service for different use cases.
- Apply analytical skills to cloud resource management through cost and performance analysis.
- Integrate the knowledge acquired in previous weeks to prepare high-quality datasets for visualization.

---

**Tasks to Be Completed This Week**

| Day | Tasks | Date |
|---|---|---|
| **Monday** | Study AWS analytics services and compare Amazon Athena, Amazon EMR, and Amazon Redshift to understand their features, advantages, and ideal use cases for different data processing requirements. | May 11 |
| **Tuesday** | Practice building reports with Amazon QuickSight by connecting to data stored in Amazon S3 and Amazon Athena. Create datasets, design interactive visualizations, and publish dashboards for data analysis. | May 12 |
| **Wednesday** | Perform AWS cost analysis using Cost and Usage Reports. Utilize visualization tools to monitor resource consumption and optimize the project budget. | May 13 |
| **Thursday** | Practice infrastructure monitoring with Amazon CloudWatch and Grafana. Integrate performance metrics into dashboards to monitor data pipelines and server health in real time. | May 14 |
| **Friday** | Study AWS cost and resource management by organizing resources with Tags and Resource Groups. Analyze resource utilization to optimize service configurations based on actual workload requirements. | May 15 |


**Weekly Results**

- **Monday:**
  - **Achievement:** Successfully distinguished the appropriate use cases for each AWS analytics service and designed a high-level data flow from the Data Lake to analytics platforms.
  - **Lesson Learned:** Selecting the right analytics service not only improves processing performance but also has a significant impact on cost optimization for datasets of different scales.

- **Tuesday:**
  - **Achievement:** Successfully built a complete interactive data analytics dashboard with filtering capabilities and learned how to manage dashboard access permissions.
  - **Lesson Learned:** Amazon QuickSight transforms raw SQL query results into meaningful visual insights that support business decision-making.

- **Wednesday:**
  - **Achievement:** Created visual reports to monitor cloud spending over time and analyze costs across different AWS services.
  - **Lesson Learned:** Continuous cost monitoring helps ensure that cloud projects remain within budget and prevents unexpected service interruptions caused by overspending.

- **Thursday:**
  - **Achievement:** Successfully deployed a centralized monitoring solution and built dashboards to visualize key performance metrics such as CPU utilization, memory usage, and service latency.
  - **Lesson Learned:** Advanced monitoring enables early detection of performance bottlenecks in data processing pipelines before they affect system reliability.

- **Friday:**
  - **Achievement:** Applied a consistent tagging strategy across the entire analytics infrastructure and completed a resource utilization assessment for the week.
  - **Lesson Learned:** Proper resource tagging simplifies cost allocation and enables more accurate cost analysis across projects, departments, or research teams.
