**Weekly Objectives**

- **Infrastructure as Code (IaC):** Learn how to manage AWS resources using code so that infrastructure can be reused and deployed quickly across different environments.
- **Establish CI/CD Pipelines:** Build automated deployment pipelines to continuously deliver updates for machine learning models and data processing applications.
- **Strengthen Data Security:** Apply advanced encryption methods and advanced access management techniques to protect data assets and machine learning models.
- **Optimize Cost and Performance:** Learn how to choose cost-effective pricing plans and optimize cloud resources to maximize performance while minimizing operational costs.

---

**Tasks to be completed this week:**

| Day | Task | Date |
|---|---|---|
| Monday | Study and practice Infrastructure as Code (IaC) using AWS CloudFormation and AWS CDK by defining resources such as Amazon S3, Amazon EC2, and Amazon SageMaker to automate infrastructure deployment. | 25/5 |
| Tuesday | Practice system administration using AWS Systems Manager and Session Manager while utilizing Resource Groups and Resource Tags to organize cloud resources and manage access permissions. | 26/5 |
| Wednesday | Practice data security using AWS KMS and AWS Secrets Manager while applying Amazon S3 Security Best Practices to protect the Data Lake from unauthorized access. | 27/5 |
| Thursday | Practice building CI/CD pipelines with AWS CodePipeline and use AWS Step Functions to automate and orchestrate machine learning training and deployment workflows. | 28/5 |
| Friday | Study AWS Savings Plans and Reserved Instances while practicing Amazon EC2 right-sizing to reduce costs and improve resource utilization. | 29/5 |

**What were the achievements this week?**

- **Monday:**
  - **Achievement:** Successfully deployed a complex infrastructure stack using a single template and gained a solid understanding of infrastructure version management.
  - **Lesson Learned:** Using Infrastructure as Code (IaC) eliminates errors caused by manual configuration and enables environments to be replicated within minutes instead of being rebuilt from scratch.

- **Tuesday:**
  - **Achievement:** Successfully managed cloud instances remotely through a web browser and established a well-organized resource classification system for different research projects.
  - **Lesson Learned:** Effective system administration improves security and makes monitoring cloud resources much easier as projects continue to grow.

- **Wednesday:**
  - **Achievement:** Successfully encrypted sensitive data and connection credentials and configured automatic secret rotation for the system.
  - **Lesson Learned:** Data encryption and security are essential for protecting machine learning models and customer information.

- **Thursday:**
  - **Achievement:** Successfully established a basic MLOps workflow in which models are automatically updated whenever changes are made to the source code or datasets.
  - **Lesson Learned:** Automating deployment workflows reduces waiting time and ensures that production models are always the latest and most reliable versions.

- **Friday:**
  - **Achievement:** Completed a cloud cost optimization report.
  - **Lesson Learned:** A well-designed cloud system should not only operate reliably but also be cost-optimized to ensure efficient use of computing resources.
