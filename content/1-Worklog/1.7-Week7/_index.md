> **Duration:** 01/06/2026 - 06/06/2026 Research on Customer Behavior Analytics and System Architecture Design

---

### Weekly Objectives

- Study the Customer Behavior Analytics problem in the e-commerce domain, including business objectives, input data, and key analytical metrics.
- Explore the dataset and identify the data sources required for building a customer behavior analytics system.
- Analyze the system requirements and identify the core architectural components.
- Design the overall architecture for a customer behavior analytics system on the AWS Cloud.
- Define the role of each AWS service within the data pipeline, from data ingestion and storage to processing and visualization.

---

### Work Activities

- Conduct research on Customer Behavior Analytics and its applications in analyzing user behavior on e-commerce platforms.
- Explore the **E-commerce Transactions & Clickstream** dataset, understand its schema, relationships among tables, and key attributes used for analytics.
- Define the end-to-end data flow, from user-generated events to data processing and visualization on dashboards.
- Analyze and select appropriate AWS services for the system architecture, including **Amazon Kinesis Data Streams**, **AWS Lambda**, **Amazon S3**, **AWS Glue**, **Amazon Athena**, and **Streamlit**.
- Design the overall system architecture based on the Data Lake architecture pattern, describing the data flow between components and the responsibilities of each AWS service.
- Create an **Architecture Diagram** that serves as the blueprint for implementation in the following weeks.

---

### Achievements

- Gained a comprehensive understanding of the Customer Behavior Analytics problem and identified the key analytical objectives for an e-commerce platform.
- Understood the structure and characteristics of the dataset, identifying the primary data tables required for customer behavior analysis.
- Completed the overall architecture design for the AWS-based customer behavior analytics system.
- Clearly defined the responsibilities and interactions of AWS services throughout the entire data processing pipeline.
- Developed a comprehensive Architecture Diagram illustrating the complete workflow, from data ingestion and storage to processing, analytics, and visualization.
