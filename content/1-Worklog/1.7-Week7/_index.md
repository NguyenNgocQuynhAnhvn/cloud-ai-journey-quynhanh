**Weekly Objectives**

- Study the Customer Behavior Analytics problem in the e-commerce domain, including business objectives, input data, and key analytical metrics.
- Explore the dataset and identify the data sources required for building a customer behavior analytics system.
- Analyze the system requirements and identify the essential components of the architecture.
- Design the overall architecture for the customer behavior analytics system on AWS.
- Define the role of each AWS service throughout the pipeline, from data ingestion and storage to processing and data visualization.

---

**Tasks to be completed this week:**

| Day | Task | Date |
|---|---|---|
| Monday | Study the fundamentals of Customer Behavior Analytics, explore its applications in e-commerce, and identify the project's analytical objectives. | 1/6 |
| Tuesday | Explore the **E-commerce Transactions & Clickstream** dataset, analyze its structure, understand the relationships between tables, and identify the data fields required for processing and analysis. | 2/6 |
| Wednesday | Analyze the data flow from user-generated events to data processing and dashboard visualization. Research and evaluate suitable AWS services for building the system. | 3/6 |
| Thursday | Design the overall system architecture based on the Data Lake model and define the roles of Amazon Kinesis Data Streams, AWS Lambda, Amazon S3, AWS Glue, AWS Athena, and Streamlit throughout the data processing pipeline. | 4/6 |
| Friday | Finalize the Architecture Diagram, review the entire system design, and prepare documentation as the foundation for the implementation phase in the following weeks. | 5/6 |

**What were the achievements this week?**

- **Monday:**
  - **Achievement:** Gained a solid understanding of the Customer Behavior Analytics problem and identified the analytical objectives and key metrics for evaluating customer behavior.
  - **Lesson Learned:** Defining clear analytical objectives at the beginning provides a solid foundation for designing the architecture and selecting appropriate data throughout the project.

- **Tuesday:**
  - **Achievement:** Completed the dataset exploration, understood the data structure, and identified the relationships between tables required for system development.
  - **Lesson Learned:** A thorough understanding of the dataset enables the design of an efficient data processing pipeline while reducing implementation errors in later stages.

- **Wednesday:**
  - **Achievement:** Identified the system's data flow and selected appropriate AWS services to support data ingestion, storage, processing, and analytics.
  - **Lesson Learned:** Choosing the right AWS services helps ensure high system performance, scalability, and cost efficiency.

- **Thursday:**
  - **Achievement:** Completed the overall system architecture design and clearly defined the role of each AWS service throughout the data processing pipeline.
  - **Lesson Learned:** A well-designed architecture facilitates system implementation and provides a scalable foundation for future development.

- **Friday:**
  - **Achievement:** Completed the Architecture Diagram illustrating the end-to-end workflow from data ingestion, storage, and processing to analytics and data visualization.
  - **Lesson Learned:** The architecture diagram serves as an essential reference for aligning system implementation and supporting future development, operation, and maintenance.
